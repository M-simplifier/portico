# Portico Roadmap

## Phase 0: Foundation

- freeze the initial domain statement around published static sites
- settle the semantic core/theme split
- keep the repository public-facing from the start
- document the difference between Portico and Asterism clearly

## Phase 1: First Working Slice

- grow the block vocabulary beyond the initial skeleton
- implement HTML document rendering
- emit multi-page static output
- ship one official theme that already looks intentional
- build one example official site inside the repo

## Phase 2: Static-Site Practicality

- add canonical plus OG/Twitter metadata helpers
- add a first validation/diagnostic layer for publishability checks
- add GitHub Pages-ready build output such as `404.html`, `robots.txt`, and `sitemap.xml`
- support navigation, collections, article pages, and docs pages well
- support code samples, callouts, feature grids, link grids, and release surfaces
- introduce a small asset pipeline story

## Phase 3: Controlled Escape Hatches

- support custom blocks without collapsing back into raw markup everywhere
- define the boundary for light interaction and optional islands
- clarify future interop with Asterism where it makes sense

## Phase 4: Beta Prep

- define and publish the public-alpha gate explicitly
- freeze the supported public surface
- decide the registry package split
- strengthen agent-oriented docs
- add verification around real example output
